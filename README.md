# 🌐 C-TCP-Server
A lightweight, concurrent, and object-oriented TCP Server library written entirely in standard C. 

This library abstracts away the heavy boilerplate of POSIX socket management, byte-order translation, and multiprocess forking, allowing developers to spin up a robust background TCP server in just a few lines of code.

## ✨ Features
* **OOP-Style API in C:** Uses custom structs and function pointers to simulate object-oriented methods (e.g., `serve->listen()`, `conn->send()`).
* **Concurrent Architecture:** Automatically handles multiple simultaneous clients using a double-`fork()` daemon process model. 
* **Safe Memory Management:** Built with strict memory hygiene, utilizing `memset` structures, safe string boundary copying (`strncpy`), and protection against wild pointers.
* **Zero-Boilerplate Event Loop:** Developers only need to write a single callback function to handle incoming data; the library manages the file descriptors, endianness translation (`htons`/`ntohs`), and connection lifecycle.

## 🚀 Quick Start
Because the library handles the underlying POSIX systems, initializing a server and reading/writing data is incredibly simple.

```c
#include "tcpserver.h"

// 1. Write your custom handler function
private void handler(connection *conn) {
    char buf[128];
    memset(buf, 0, 128);
    
    // The library automatically targets the correct client socket
    conn->send("Welcome to the Server!\n", 23);
    conn->recv(buf, 127);
    
    printf("Client says: '%s'\n", buf);
}

int main() {
    // 2. Initialize the server on your chosen port
    server *serv = tcpserver(8585, handler);
    
    if (serv && !serv->ok()) {
        printf("Error: %s\n", serv->err());
        return -1;
    }
     
    printf("Listening on port 8585...\n");
    
    // 3. Start the event loop (forks into the background)
    return serv->listen(); 
}
